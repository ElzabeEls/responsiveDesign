Key Takeaways

  • declare that images, videos, iframes should never be rendered at a size wider than their containing element using max-inline-size.
  • img {
    max-inline-size: 100%;
    block-size: auto;
    }
  • * max-inline-size is a local property
  • Add a block-size value of auto to retain aspect-ratio.
  • use object-fit: contain and the aspect-ratio properties together to retain the aspect ratio and prevent stretching even if that means leaving white space
  • use object-fit:cover and the aspect-ratio properties together to preserve the aspect ratio and prevent stretching even if it means cropping the image.
  • use object-position property to adjust the crop: object-position: top center;
  • add width and height atributes to img tag. This tells the browser the ratio and will prevent content from jumping around when image loads.
  • add loading="lazy" to images below the fold to only load those images once user scrolls there.
  • loading="eager" prevents default lazy load
  • Prioritise loading of main images with fetchpriority="high" . Only use this if image is VERY important
  • rel="preload" fethces images added via CSS or JS ahead of time.
  • using imagesrcset and imagesizes are more advanced and is not supported by all browsers. Exclude href fallback when using this.
  • adding the decoding="async" attribute tells the browser that the image can be decoded asynchronously.
  • use the sync value if the image itself is the most important piece of content to prioritize: decoding="sync"
  • The decoding attribute influences whether the browser waits for image decoding before rendering other content.
  • Setting 'sync' on important images can make the browser wait for both image decoding and content rendering, displaying them together.
  • Setting 'async' can allow content to be displayed faster without waiting for image decoding.
  • you can use srcset to switch between the same image of different sizes so that people viewing on mobile don't download unnecessarilty large images: srcset="small-image.png 300w, medium-image.png 600w, large-image.png 1200w" You still need to have a valid src attribute, but now the browser can replace its value with one of the options listed in the srcset attribute.
  • must use the sizes attribute with the width descriptor. tells the browser what size you expect the image to be displayed under different conditions: sizes="(min-width: 66em) 33vw, (min-width: 44em) 50vw, 100vw"
  • Use the density descriptor to describe the pixel density of the image in relationship to the image in the src attribute: srcset="small-image.png 1x, medium-image.png 2x, large-image.png 3x". This keeps images looking sharp.
  • Use an empty alt attribute to convey to screen readers if an image is just presentational.
  • Ideally, your presentational or decorative images shouldn't be in your HTML.
  • You can specify multiple image candidates using the image-set function for background-image: background-image: image-set( small-image.png 1x, medium-image.png 2x, large-image.png 3x ); The browser will choose the most appropriate image for the device's pixel density. .
