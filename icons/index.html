Key Takeaways

  • PNGs (and JPGs, WebP, and AVIF) are bitmap images. Bitmap images store information as pixels.
  • In an SVG, information is stored as drawing instructions. These instructions are relative.
  • XML is used to write the instructions in an SVG file.
  • If you put the SVG inside your HTML, use the aria-hidden attribute to hide it from assistive technology.
  • Usually the alt attribute describes the contents of the image (“Three horizontal lines.”) but with standalone icons, the alt attribute describes the meaning of the image (“Menu”).
  • standalone icons are not presentational an should not be inserted via CSS
  • Another option is to put the accessible name on the link or button itself and declare that the image is presentational. Use the aria-label attribute to provide the accessible name.
  • If you put the SVG inside your HTML, use the aria-label attribute on the link or button to give it an accessible name and use the aria-hidden attribute on the icon.
  • SVGs don't have alt tags
  • SVGs can my styled more extensively
  • If you need to style SVGs that are background images in your CSS, read Una's article on colorizing SVG backgrounds.
  • Sara Soueidan has written about accessible icon buttons.
  • If you're using a graphic design tool SVGOMG to export SVGs, use to optimize the output.
